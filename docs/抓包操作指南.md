# xyzw_analyzer 抓包操作指南

## 架构概览

```
[游戏客户端/H5浏览器]
        │  WiFi代理 / VPN
        ▼
[MITM代理 :12311]  ←─── game-mitm 库，TLS中间人
        │  解密后转发
        ▼
[游戏服务器 xxz-xyzw.hortorgames.com]
        │
        │  捕获到的协议数据
        ▼
[Web UI :12582]  ←─── WebSocket实时推流，浏览器查看
```

## 三种抓包方式对比

| 方式 | 是否需要root | 适用场景 | 额外部署 |
|------|------------|---------|---------|
| **H5网页版（推荐）** | 否 | 电脑浏览器 | 无 |
| WiFi代理 + 手机 | 需要（Android 7+） | 原生App | 无 |
| WireGuard VPN透明代理 | 需要 | 原生App | 需运行setup-vpn.sh |

> **Android 7+ 原生App不信任用户级CA证书**，只有系统级证书才被信任，所以原生App抓包需要root。
> H5网页版运行在浏览器中，浏览器信任用户安装的CA证书，因此无需root。

## 推荐方式：H5网页版（无需root）

### 前提条件

- 服务已部署，容器 `xyzw-analyzer` 正在运行
- 阿里云安全组已放行 TCP `12311`、`12582`
- 有游戏账号（小程序/QQ登录）

### 第一步：确认服务状态

```bash
ssh root@47.120.18.209 "docker ps | grep xyzw-analyzer"
ssh root@47.120.18.209 "ss -tlnp | grep -E '12311|12582'"
```

两个端口都有监听才说明服务正常。

### 第二步：电脑安装 MITM CA 证书

浏览器打开 `http://47.120.18.209:12582/cert` 下载证书文件。

**Windows：**
1. 双击证书文件
2. 选"本地计算机" → 下一步
3. 选"将所有证书放入以下存储" → 浏览 → 选"受信任的根证书颁发机构"
4. 完成

**Mac：**
1. 双击证书，导入到"系统"钥匙串
2. 找到该证书 → 右键"显示简介"
3. 展开"信任" → "使用此证书时"改为"始终信任"

**Linux（Chrome）：**
```bash
certutil -d sql:$HOME/.pki/nssdb -A -t "CT,," -n "xyzw-ca" -i /path/to/cert.crt
```

### 第三步：电脑浏览器配置代理

**推荐：Chrome/Edge 安装 Proxy SwitchyOmega 插件**
- 新建情景模式
- 代理协议：HTTP
- 代理服务器：`47.120.18.209`
- 端口：`12311`
- 切换到该情景模式

**或者：Windows 系统代理**
- 设置 → 网络和Internet → 代理 → 手动设置代理
- 地址：`47.120.18.209`，端口：`12311`

### 第四步：首次登录捕获 AuthData

这是**关键步骤**，只需做一次，目的是让代理记录登录凭证。

确保浏览器已走代理，然后访问游戏H5地址：
```
https://xxz-xyzw-res.hortorgames.com/h5web/index.html
```

用小程序/QQ账号正常登录游戏。登录请求（`/login/authuser`）会经过MITM代理，代理自动将 `AuthData` 保存到内存并持久化到 `data/auth_data.txt`，**容器重启后自动恢复，无需重新登录**。

> **原理（capture.go）：**
> ```go
> if ctx.Req.URL.Path == "/login/authuser" {
>     AuthData = hex.EncodeToString(body)  // 保存登录凭证（内存+文件）
>     onAuthData(AuthData)                 // 触发持久化回调
>     return body
> }
> ```

### 第五步：使用"网页登录"按钮

1. 打开 Web UI：`http://47.120.18.209:12582`
2. 确认右上角显示"已连接"
3. 点击"**网页登录**"按钮

后端将 `Redirect = true`，然后自动打开H5游戏。下次H5游戏调用 `/login/authuser` 时，代理用保存的 AuthData 替换请求体，实现自动登录。

Web UI 开始实时显示游戏协议数据。

### 注意事项

- AuthData 现在**持久化到文件**（`data/auth_data.txt`），容器重启后自动恢复，**无需重新登录**
- 每次想分析新的协议数据，点一次"网页登录"即可（AuthData已有则无需重复第四步）
- 若点"网页登录"提示"请先使用小程序/QQ登录"，说明AuthData为空，需重做第四步

### 通过 web_helper 同步凭证（免登录方案）

如果已经在 **xyzw_web_helper** 中导入了该账号的 bin 文件，可以直接将凭证推送给分析器，无需重新登录游戏：

1. 打开 web_helper 的**批量日常任务**页面
2. 展开顶部"🔗 同步到分析器"面板
3. 填写分析器地址（默认 `http://localhost:12582`，跨容器用 `http://xyzw-analyzer:12582`）
4. 点"**检测**"确认分析器在线
5. 选择要同步的账号 → 点"**同步凭证**"
6. 提示"已成功同步"后，直接在 Web UI 点"网页登录"即可

## 备用方式：WiFi代理（需手机root）

适合需要抓原生App数据的场景。

### 前提：手机需要root（Magisk）

Android 7+ 原生App只信任系统级CA证书。需用 Magisk 模块提升证书级别：
```
Magisk → 模块 → 搜索 "Move Certificates" → 安装 → 重启
```

### 步骤

1. 手机浏览器访问 `http://47.120.18.209:12582/cert` → 下载安装CA证书
2. Magisk Move Certificates 将其提升为系统证书
3. 手机 WiFi → 代理 → 手动 → 主机 `47.120.18.209`，端口 `12311`
4. 打开游戏，Web UI 查看数据

## 备用方式：WireGuard VPN透明代理（需手机root）

同样需要系统级证书，root要求与WiFi代理一致。

**服务器部署（本机执行一次）：**
```bash
cd /work/code/tongli/xyzw/xyzw_analyzer
sh deploy/setup-vpn.sh
```

阿里云安全组放行 UDP `51820`。

**手机端：**
1. 安装 WireGuard App
2. 扫描脚本输出的二维码（或用 `xyzw.conf.png`）
3. 连接VPN
4. 安装系统级CA证书（同WiFi代理方式）

## 常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| Web UI 无法访问 | 12582端口未开放 | 阿里云安全组放行TCP 12582 |
| 配置代理后网络断 | 12311端口未开放 | 阿里云安全组放行TCP 12311 |
| 游戏黑屏卡住 | CA证书未被App信任（Android 7+） | 换H5方式，或root后提升证书 |
| "请先使用小程序/QQ登录" | AuthData为空 | 先走第四步登录一次 |
| 容器重启后"网页登录"失效 | data/auth_data.txt 不存在或为空 | 走第四步登录一次或从web_helper同步 |
| H5游戏加载失败 | 浏览器代理或CA证书未配置 | 检查代理设置和证书安装 |
